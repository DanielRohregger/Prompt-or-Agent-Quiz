Funktionen, Grenzen und wann du was nimmst – je Punkt aufklappen. Jeweils mit offizieller Microsoft-Quelle.

Einfacher Prompt · Microsoft 365 Copilot Chat

✅ Funktionen

  • Sofort-Antworten in Word, Excel, Outlook, Teams und im Copilot-Chat
  • Nutzt deine Arbeitsinhalte über Microsoft Graph (nur was du sehen darfst) plus Web
  • Zusammenfassen, Entwerfen, Umformulieren, Q&A – schnell in Sekunden

⛔ Grenzen

  • Einmalig pro Konversation – nichts wird als wiederverwendbarer Agent gespeichert
  • Keine Aktionen in Fremdsystemen; für schnelle Aufgaben, nicht für tiefe mehrstufige Analysen

🎯 Wann

  • Einmalige Aufgabe, der Kontext liegt bereits im Chat

🔗 Quelle: learn.microsoft.com – Was ist Microsoft 365 Copilot?

Researcher-Agent (fertig in M365 Copilot)

✅ Funktionen

  • Intelligenter Assistent für komplexe, mehrstufige Recherche – einfach unter „Agents" auswählen
  • Kombiniert Web UND Arbeitsinhalte (Dateien, E-Mails, Meetings, Chats) zu einem quellenzitierten Bericht
  • Suchbereich wählbar (nur Web / nur Arbeit / beides); optional Connectoren zu Salesforce, ServiceNow, Confluence

⛔ Grenzen

  • Braucht bewusst mehr Zeit (Minuten statt Sekunden)
  • Für Recherche & Synthese – nicht zum Rechnen auf Rohdaten (→ Analyst); M365-Copilot-Lizenz nötig

🎯 Wann

  • Markt-, Wettbewerbs- oder Regulatorik-Analyse mit belegten Quellen

🔗 Quelle: learn.microsoft.com – Researcher-Agent in Microsoft 365 Copilot

Analyst-Agent (fertig in M365 Copilot)

✅ Funktionen

  • Virtueller Datenanalyst für strukturierte Daten (Excel, CSV, TSV)
  • Generiert und führt im Hintergrund Python-Code aus – transparent sichtbar
  • Rechnet Trends, Ausreißer und Prognosen, erstellt Visualisierungen, auch dateiübergreifend

⛔ Grenzen

  • Für quantitative Daten – nicht für Web-Recherche (→ Researcher)
  • Qualität hängt von der Datenqualität ab; M365-Copilot-Lizenz nötig

🎯 Wann

  • Echte Berechnungen und Diagramme aus Tabellen statt „Raten aus dem Text"

🔗 Quelle: learn.microsoft.com – Analyze and visualize data with the Analyst agent

Copilot Cowork · der agentische KI-Kollege

✅ Funktionen

  • Agentischer „KI-Kollege" in Microsoft 365 Copilot (allgemein verfügbar) – erledigt Aufgaben in deinem Namen
  • Bearbeitet mehrstufige, lang laufende Aufgaben eigenständig über mehrere Tools und liefert ein fertiges Ergebnis statt eines Entwurfs
  • Kann Mails senden, Meetings planen, Dokumente erstellen (Word, Excel, PowerPoint, PDF), in Teams posten, Kalender verwalten, Tages-Briefings erstellen, org-weit suchen, Deep Research betreiben; Prompts sind planbar (wiederkehrend)

⛔ Grenzen

  • Arbeitet in deinem M365-Rahmen mit deinen Berechtigungen
  • Allgemeiner persönlicher Assistent – kein Ersatz für einen gebauten, in Fremdsysteme integrierten Prozess-Agenten (→ Copilot Studio); M365-Copilot-Lizenz nötig

🎯 Wann

  • Eine Aufgabe soll end-to-end erledigt und ein fertiges Ergebnis gesendet/abgelegt werden – ohne etwas zu bauen

🔗 Quelle: learn.microsoft.com – Copilot Cowork (häufige Fragen)

Agent Builder · No-Code in M365 Copilot

✅ Funktionen

  • Deklarative Agents ohne Code direkt in M365 Copilot (Teams, microsoft365.com/chat)
  • Eigene Anweisungen, Tonalität und Starter-Prompts; im Unternehmen teilbar
  • Wissensquellen: SharePoint-Dateien/-Ordner/-Sites, bis zu 50 OneDrive-Dateien, bis zu 4 öffentliche Websites, bis zu 5 Teams-Chat-URLs, hochgeladene Dateien, M365-Copilot-Connectoren (falls vom Admin aktiviert)

⛔ Grenzen

  • Keine echten Aktionen in Fremdsystemen, keine Auslöser/Autonomie
  • Für einfache, geradlinige Szenarien; begrenzte Anzahl an Quellen

🎯 Wann

  • Wiederkehrender Team-Assistent auf M365-Wissen (+ etwas Web), No-Code

🔗 Quelle: learn.microsoft.com – Agent Builder · Wissensquellen & Limits

SharePoint Agent · Q&A auf einer Site

✅ Funktionen

  • Beantwortet Fragen zu den Inhalten EINER SharePoint-Site oder -Bibliothek
  • Jede Site hat einen fertigen Agent; Owner/Member bauen in wenigen Klicks eigene (Umfang, Identität, Verhalten, Startprompts) – ohne IT
  • Erbt automatisch SharePoint-Berechtigungen und Vertraulichkeitslabels; per Link oder in Teams-Chat teilbar

⛔ Grenzen

  • Nur SharePoint-Inhalte – kein Web, keine externen Systeme, keine Aktionen/Automatisierung
  • Tipp aus der Doku: Inhalte klein & kuratiert halten für gute Antworten; M365-Copilot-Lizenz oder Pay-as-you-go nötig

🎯 Wann

  • Dokumenten-Q&A über eine bestimmte Site/Bibliothek, schnell durch den Content-Owner

🔗 Quelle: support.microsoft.com – Erste Schritte mit Agents in SharePoint

Copilot Studio · Low-Code, Aktionen & Autonomie

✅ Funktionen

  • Grafisches Low-Code-Tool für Agents und Agent-Flows (copilotstudio.microsoft.com)
  • Topics + generative Orchestrierung; vorgefertigte und eigene Connectoren zu vielen Datenquellen
  • Aktionen & Automatisierung, Auslöser und autonome Agents (reagieren auf Ereignisse, laufen im Hintergrund)
  • Veröffentlichung über viele Kanäle: Website, Teams, Facebook, Apps, Azure Bot Service

⛔ Grenzen

  • Kein eigenes Modell-Training / Fine-Tuning
  • Sehr komplexe, produktreife KI-Systeme brauchen Pro-Code (→ Foundry); Governance/Umgebungen mitdenken

🎯 Wann

  • Prozess-Automatisierung, Fremdsystem-Anbindung, Multi-Channel-Bots, Autonomie

🔗 Quelle: learn.microsoft.com – Overview Microsoft Copilot Studio

Microsoft Foundry · Pro-Code für Entwickler

✅ Funktionen

  • Verwaltete Entwickler-Plattform zum Bauen, Bereitstellen und Skalieren von Agents
  • Freie Modellwahl aus dem Foundry-Katalog (z. B. GPT, Llama, DeepSeek); Prompt-Agents (konfiguriert) und Hosted-Agents (eigener Code als Container)
  • Tools: Web-/Datei-Suche, Code-Interpreter, Memory, MCP-Server; RAG mit Azure AI Search; Multi-Agent-Orchestrierung
  • Beobachtbarkeit/Tracing, Entra-Identität, RBAC, Content-Filter; SDKs für Python, C#, JavaScript, Java

⛔ Grenzen

  • Braucht Entwickler-Know-how, Azure und Governance
  • Overkill für einfache Wissens-Agents; Kosten- und Betriebsverantwortung liegt bei dir

🎯 Wann

  • Eigene Modelle/Fine-Tuning, Skalierung auf Millionen Anfragen, tiefe Custom-Architektur

🔗 Quelle: learn.microsoft.com – What is Microsoft Foundry Agent Service?
